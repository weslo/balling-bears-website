# Agent Instructions

## Project Brief

Balling Bears is a website for the Balling Bears World of Warcraft raiding guild on Mal'Ganis. The site is built with TypeScript, React, and Vite, and should stay lightweight, easy to update, and easy to run locally.

## Working Style

- Keep code readable and practical. Prefer straightforward solutions over clever ones.
- Comment code when it helps the next person understand intent or a non-obvious decision.
- Keep comments concise and clear. Avoid unnecessary jargon, acronyms, or filler.
- Do not add comments for code that is already obvious from the implementation.

## Git Workflow

- Commit changes at regular intervals when a meaningful unit of work is complete.
- Use clear, concise commit messages that explain what changed.
- Push changes regularly so the remote history stays current and useful.
- Keep history easy to scan by avoiding vague messages like `updates` or `fix stuff`.

## File Hygiene

- Do not leave empty directories behind when removing files.
- Keep the repo tidy and remove generated artifacts that should not be committed.

## README Expectations

- Update `README.md` whenever product behavior, setup steps, or operational workflow changes.
- Keep README instructions current, clear, and easy for a teammate to follow.
- Make sure the README explains how to install dependencies, run the site locally, build it, and continue iterating safely.
